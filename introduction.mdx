  <h2 className="text-2xl sm:text-3xl font-semibold text-gray-900 dark:text-white mb-3">
    What is preload-rs?
  </h2>
  <p className="text-base text-gray-600 dark:text-gray-400 mb-8">
    preload-rs is an adaptive daemon that monitors which programs you run and prefetches their binaries and shared libraries into RAM before you launch them. Using a Markov-based probabilistic model, it learns correlations between applications and predicts what you'll need next.
  </p>

  <CardGroup cols={2}>
    <Card title="Smart Prefetching" icon="brain" href="/guides/understanding-preload">
      Uses Markov chains to predict which applications you'll launch next based on your usage patterns
    </Card>
    <Card title="Memory Aware" icon="memory" href="/guides/tuning-performance">
      Dynamically adjusts prefetch budgets based on available system memory to avoid performance degradation
    </Card>
    <Card title="Process Monitoring" icon="chart-line" href="/architecture/overview">
      Continuously scans running processes and tracks mapped files to build accurate usage models
    </Card>
    <Card title="Persistent State" icon="database" href="/architecture/persistence">
      Saves learned patterns to SQLite so predictions improve over time across system reboots
    </Card>
  </CardGroup>
</div>

<div className="mt-16 mb-16 max-w-5xl mx-auto px-6">
  <h2 className="text-2xl sm:text-3xl font-semibold text-gray-900 dark:text-white mb-3">
    Quick Start
  </h2>
  <p className="text-base text-gray-600 dark:text-gray-400 mb-8">
    Get preload-rs running on your Linux system in three simple steps
  </p>

  <Steps>
    <Step title="Install preload-rs">
      Clone the repository and build with Cargo:
      
      ```bash
      git clone https://github.com/arunanshub/preload-rs.git
      cd preload-rs
      cargo build --release
      ```
    </Step>
    
    <Step title="Create a configuration file">
      Copy the example configuration and customize it for your system:
      
      ```bash
      cp docs/config.example.toml config.toml
      ```
      
      <Accordion title="View example configuration">
        ```toml config.toml
        [model]
        cycle = 20
        use_correlation = true
        minsize = 2000000
        active_window = 21600
        decay = 0.01
        
        [model.memory]
        memfree = 50
        memcached = 0
        
        [system]
        doscan = true
        dopredict = true
        sortstrategy = "block"
        ```
      </Accordion>
    </Step>
    
    <Step title="Run the daemon">
      Start preload-rs with your configuration:
      
      ```bash
      ./target/release/cli --config config.toml -v
      ```
      
      <Info>
        The daemon will begin monitoring your system immediately. Run with `-v`, `-vv`, or `-vvv` for increasing verbosity levels.
      </Info>
    </Step>
  </Steps>
</div>
